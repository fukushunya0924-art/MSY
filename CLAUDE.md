# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 概要

黒潮大蛇行（LM/NLM）レジームが日本近海4魚種の資源量に与える影響を、捕食被食ODEモデルとパラメータ推定で定量化する研究コード。

- **被食者 (x)**: マイワシ (x1), カタクチイワシ (x2)
- **捕食者 (y)**: ブリ (y1), サワラ (y2)
- **レジーム分割**: NLM 2006–2016 / LM 2017–2024
- **旧捕食者**: ヤリイカ・スルメイカ（Phase 2 で変更済み）

## 実行コマンド

環境は `archive/6.8_old/` 内の Poetry で管理されている。

```bash
# 依存パッケージのインストール（archive/6.8_old/ ディレクトリ内）
cd archive/6.8_old && poetry install

# 標準推定（src/ から実行）
cd src && python3 run_estimation.py            # full(16変数)モデル
cd src && python3 run_estimation.py capacity   # capacity(10変数)モデル
cd src && python3 run_estimation.py capacity_ry  # 12変数モデル

# 1位組み合わせ（ヤリイカ+スルメイカ / ブリ+サワラ）の推定
cd src/rank1_squid_largefish && python3 run_rank1.py [full|capacity|capacity_ry]

# 全魚種の組み合わせスキャン
cd src && python3 scan_species_combinations.py

# データ確認
cd src && python3 data_loader.py
```

出力 PNG は `outputs/` に保存される（例: `outputs/fit_full.png`）。

### ディレクトリ構成

```

src/          現行コード（モジュール分割版）

  model.py                ODE定義・推定エンジン（estimate関数）
  run_estimation.py       ベースライン vs 改善版の比較実行

  rank1_squid_largefish/  scan で1位になった組み合わせ専用の実行スクリプト
outputs/      推定結果の PNG（fit_*.png）
archive/      試行版・旧バージョン
  6.8_old/    旧バージョン（単一スクリプト、poetry環境定義を持つ）

```

### 推定の流れ（`src/`）

1. `data_loader.load_clean_dataframe()` → 4種CSVをマージ、NaN除去
2. `data_loader.get_series()` → 全種を千トン統一、漁獲圧 `f = catch/biomass`（上限0.95）を計算
3. `run_estimation.regime_masks()` → NLM/LMの年次マスクで時系列を分割
4. `model.estimate()` → 各レジームをODE推定
   - 正規化空間（各種を全期間平均で除した平均1.0の空間）でODEを解く
   - `scipy.integrate.solve_ivp(method="LSODA")` で数値積分
   - `scipy.optimize.least_squares(method="trf")` で対数誤差を最小化
   - マルチスタート（`n_starts`個の初期値）で局所解を回避
   - 相互作用パラメータに L2 正則化（`reg_lambda`）を付与可
5. 推定後に `model._to_absolute()` で元スケールの物理パラメータへ換算

### モデル種別（`model.MODELS`）

| モデル | パラメータ数 | 特徴 | 採用方針 |
|---|---|---|---|
| `capacity` | 10変数 | r_y固定（r_y1=0.3, r_y2=0.4）、種内競争なし | 使用可 |
| `capacity_ry` | 12変数 | r_y自由化、種内競争なし | **主力モデル** |
| `full` | 16変数 | r_y自由化 + 種内競争項（A_x, A_y）あり | **使用しない** |

> **方針（2026-06-13決定）**: 今後は**環境収容力なし（種内競争項 α を含まない）モデルのみ**で進める。
> すなわち `capacity`（10変数）と `capacity_ry`（12変数）のみ対象。
> `full`（16変数）は LM 期 8点に対して過剰パラメータ・識別性崩壊・非物理的パラメータ（c1=41等）が生じるため除外。

### データスケール換算

各CSVで単位が異なるため `data_loader.get_series()` で千トンに統一している：
- マイワシ: 万トン × 10
- カタクチイワシ: 千トン（そのまま）/ 漁獲量はトン ÷ 1000
- ヤリイカ: トン ÷ 1000
- スルメイカ: 千トン（そのまま）

## 研究進捗ログ（2026-06-12〜13）

### Phase 1: ベースライン診断と推定精度の改善

#### 問題発見
archive/6.8_old/ の元コードで `least_squares` を**単一初期値・1回**だけ呼ぶと、被食者2種（マイワシ・カタクチ）だけ合わせ捕食者2種（ヤリイカ・スルメイカ）が R²<0 になる局所解に落ちることが判明した。

```
元コード ベースライン（fullモデル）
  NLM: 平均R²=-0.566  (y1=-0.357, y2=-3.733)
  LM:  平均R²=+0.167  (y1=-1.021)
```

#### 改善内容（src/ に実装）
| 改善 | 効果 |
|---|---|
| **マルチスタート最適化**（n_starts=40） | 局所解を回避。最大の効果 |
| **LSODA積分 + 許容誤差強化** | 頑健な数値積分（RK45→LSODA, rtol=1e-7） |
| **L2正則化**（reg_lambda=0.02） | 過剰適合・非識別性を緩和 |
| **実スケールR²/NRMSEの算出** | 適合度を定量的に比較可能にした |

#### 改善後の結果（src/ full モデル）
```
NLM: 平均R²=+0.500  NRMSE=0.267  (y1=+0.371, y2=-0.207)
LM:  平均R²=+0.513  NRMSE=0.235  (y1=+0.204, y2=+0.802)
```

### Phase 2: 魚種組み合わせの探索

#### 動機
LM期のカタクチイワシ(x2)とヤリイカ(y1)のフィットが悪い原因：
- **カタクチLM期**: CV=0.45、92→299と低水準での振動+急増 → ODEで追えない構造的ミスマッチ
- **ヤリイカLM期**: CPUEベースの推定量で観測誤差が大きく、上下振動が激しい

#### スキャン手法
`src/scan_species_combinations.py` で利用可能な7魚種の全有効組み合わせを評価。制約:
- **同一栄養段階グループ内**で x2種・y2種を選ぶ
- LOW（マイワシ・カタクチ）→ MID（マサバ・ヤリイカ・スルメイカ）→ HIGH（ブリ・サワラ）の順で y は x より高次

#### スキャン結果（平均CV昇順）
```
1位: ヤリイカ+スルメイカ(x) / ブリ+サワラ(y)       平均CV=0.290
2位: マサバ+ヤリイカ(x)     / ブリ+サワラ(y)       平均CV=0.310
3位: マサバ+スルメイカ(x)   / ブリ+サワラ(y)       平均CV=0.323
4位: マイワシ+カタクチ(x)   / ブリ+サワラ(y)       平均CV=0.356
現行: マイワシ+カタクチ(x)  / ヤリイカ+スルメイカ(y) 平均CV=0.472
```
ブリ(CV=0.06)・サワラ(CV=0.12)が捕食者に入るとLM期の安定性が大幅に改善する。

### Phase 3: 1位組み合わせ（ヤリイカ+スルメイカ / ブリ+サワラ）の推定

`src/rank1_squid_largefish/` に専用フォルダを作成。親の `model.py` を再利用。

#### 3モデルの比較結果（改善版・マルチスタート+正則化）

| モデル | 変数 | NLM R² | NLM NRMSE | LM R² | LM NRMSE |
|---|---|---|---|---|---|
| full | 16 | **+0.583** | **0.206** | +0.048 | 0.168 |
| capacity_ry | 12 | +0.330 | 0.244 | +0.081 | 0.163 |
| （参考）元組み合わせ full | 16 | +0.500 | 0.267 | +0.513 | 0.235 |

**NRMSEでは1位の組み合わせが両レジームとも現行より良い。**

#### 発見した構造的問題：R²とNRMSEの乖離

- LM期の R² が低い原因: ブリ(CV=0.06)はほぼ横ばいで**分散が極小** → R² = 1 - RMSE²/分散 の分母が爆発し、わずかな絶対誤差でも R² が大きな負値になる
- **平らなデータほどR²が壊れる**現象。これは適合度の問題ではなくR²の性質
- → **NRMSEを主指標として判断するのが適切**

#### 非物理的パラメータ c1 の問題

全モデルで LM 期に `c1（ブリのヤリイカ変換効率）` が異常値（full: 41.1, capacity_ry: 23.9）になる。原因：
- ブリ平均270千トン÷ヤリイカ平均10千トン = 27倍のスケール差
- 換算式 `c1 = C1 × (ブリ平均/ヤリイカ平均)` なので正規化空間での普通の値が27倍に膨らむ
- 本質的に「捕食者(ブリ)が被食者(ヤリイカ)より生物量が大きい」という生態学的逆転に起因
- 変数数の変更（16→12）では解決しない構造的問題

### Phase 5: MSY（最大持続生産量）計算

`現行コード/msy/` に MSY 計算コードを追加。4位組み合わせ（マイワシ+カタクチ / ブリ+サワラ）・`capacity_ry`(12変数) を対象。

#### 手法

既存 ODE に**定数漁獲圧ベクトル `f = [f_x1, f_x2, f_y1, f_y2]`** を注入し、積分期間 T の**平均漁獲量**（台形則 / T）を最大化する `f*` を探索する。新たな ODE は書かず `model.make_ode()` に「定数を返す callable」を渡して再利用。

- **戦略的 MSY**: レジーム全期間（NLM T=10年 / LM T=7年）を 1 本の軌道で評価。
- **戦術的 MSY**: T=1 年ごとに各年の初期値から評価。
- 探索は **4次元グリッド全評価（6⁴=1296 点）**。ブラックボックス最適化は使わず列挙。
- 漁獲率は `[0, 0.95]`（`data_loader` の上限と同じ）。正則化はレジーム別（NLM=0.0, LM=0.005）。

ファイル: `msy_core.py`（コア関数）/ `run_msy.py`（実行・コンソール出力・PNG 5枚）。

```bash
cd 現行コード/msy && python3 run_msy.py            # capacity_ry（デフォルト）
cd 現行コード/msy && python3 run_msy.py capacity   # capacity
```

#### 戦略的 MSY 結果（レジーム全期間）

| レジーム | T | MSY (千トン/年) | f* (x1, x2, y1, y2) | 主な内訳 |
|---|---|---|---|---|
| NLM | 10年 | 1,914.2 | 0.570, 0.570, **0.950**, **0.950** | カタクチ1674 / マイワシ183 / ブリ56 |
| LM | 7年 | **13,792.0** | **0.950**, 0.760, **0.950**, 0.190 | マイワシ13428 / ブリ307 / カタクチ56 |

#### 戦術的 MSY 結果（T=1年ごと）

全年・全レジームで **f* = 0.95（上限張り付き）**。

- NLM: 1,003〜1,672 千トン/年（2006・2016 が高く、中期に低下）
- LM: 2,610→5,532 千トン/年（年々増加、2022 ピーク → 2024 で 3,829 に減）

#### 重要な発見と注意点

1. **LM の戦略的 MSY が NLM の約7倍（13,792 vs 1,914）**: LM 期はマイワシが急増する局面で、ODE が長期積分（T=7年）で指数増殖的にマイワシを生産する。内訳もマイワシ 13,428（97%）に偏る。**「持続生産量」というより長期外挿のアーティファクト**の可能性が高く、解釈に注意。
2. **戦術的 MSY は全条件で f*=0.95 に張り付く**: T=1年では資源枯渇の前に「獲れるだけ獲る」が最適となり、漁獲圧と再生産のトレードオフ（=MSY らしさ）が現れない。**MSY の本質は戦略的（長期T）でのみ立ち上がる**。
3. **捕食者（ブリ y1・サワラ y2）の f* も上限張り付き傾向**（NLM 両方・LM y1 が 0.95）。ブリ・サワラは被食者ダイナミクスへの寄与が小さく「獲るほど良い」になりやすい。
4. **LM 推定の平均 R²=−1.368（NRMSE=0.177）**: R² の大きな負値は既述の「平らなデータで R² が壊れる」現象。NRMSE 0.177 は妥当な水準で矛盾しない（→ NRMSE 主体で判断）。
5. **グリッド探索の ODE 発散**: NLM は 1296 中 1042 成功（254 が発散で除外）、LM は 1284 成功。高 f 領域での発散が NLM でやや多い。

> **暫定結論**: 戦術的 MSY（短期）は f 上限張り付きで情報が乏しい。**戦略的 MSY を主指標**とするが、LM 期の巨大 MSY は外挿アーティファクト疑いがあり、積分期間 T や初期値依存性の感度確認が必要。

### 今後の課題と選択肢

> **モデル方針**: 環境収容力なし（α項なし）の `capacity`(10変数) / `capacity_ry`(12変数) のみ使用。`full` は除外。

1. **MSY の T 感度確認（最優先）**: LM 戦略的 MSY=13,792 が外挿アーティファクトか検証。積分期間 T や初期値を変えて MSY の安定性を見る。マイワシ 97% 偏重が現実的か。
2. **戦術的 MSY の扱い**: 全条件 f*=0.95 張り付きで情報が乏しい。T を 2〜3年に伸ばすか、漁獲圧上限の妥当性を再検討するか決める。
3. **1位 vs 4位の比較**: NRMSE 主体で評価し、最終的な魚種組み合わせを確定する。
4. **指標の整理**: R² と NRMSE の乖離が起きる条件を明確化し、どの指標を論文で使うか決める。
5. **パラメータ信頼区間**: `least_squares` のヤコビアンから NLM/LM のパラメータ差の統計的有意性を評価。

---

### 適合度指標の定義（再確認用）

**RMSE**: `sqrt(mean((obs - pred)^2))` — 単位は千トン。絶対誤差。  
**NRMSE**: `RMSE / mean(obs)` — 無次元。平均に対する相対誤差。魚種をまたいで比較可能。  
**R²**: `1 - SS_res/SS_tot` — 無次元。平均横線より何%改善したか。平らなデータで負に暴れるので要注意。

→ **評価はNRMSE主体、R²は補助的に使う**のが本プロジェクトでは適切。

### Phase 5a: 非減少資源量制約（持続性制約）の導入

Phase 5 の無制約MSYはf*が上限（f=0.95）に張り付き「乱獲解」に陥っていた。これを解決するため、**期末資源量が初期値の90%以上に保たれる（増えるか微減に留まる）という非減少制約**を追加。

#### 制約仕様
- **対象種**: 4種すべて（マイワシ・カタクチ・ブリ・サワラ）
- **判定方式 (mode)**: endpoint（期末のみ判定）。`B_i(T) ≥ B_i(0) × (1 − tol)` で各種の初期値割れを防ぐ。途中の自然振動は許容。
- **許容枠 (tol)**: 0.1（10%まで減少許容）。`B_i(T) ≥ B_i(0) × 0.9` を満たす点のみ feasible（持続可能）。
- **グリッド解像度**: 戦略的MSYは N_GRID=8（8^4=4096点）で精密化。

#### 戦略的MSY 結果（制約版）

| レジーム | T | 無制約MSY | **制約MSY** | 減少率 | 持続可能点 | 制約 f* |
|---|---|---|---|---|---|---|
| NLM | 10年 | 1,962.16 | **1,920.38** | −2.1% | 6 / 4096 | 0.407, 0.543, 0.950, 0.950 |
| LM | 7年 | 14,241.71 | **11,867.60** | −16.7% | 54 / 4096 | 0.950, 0.543, 0.950, 0.407 |

#### 制約MSY点での種別漁獲量と期末増減（NLM）
- マイワシ (x1): 366.7 千トン/年、期末増減 **+1787%**（爆発的増加）
- カタクチ (x2): 1,442.1 千トン/年、期末増減 **+6.3%**（律速種・ギリギリ制約）
- ブリ (y1): 110.4 千トン/年、期末増減 +188%
- サワラ (y2): 1.2 千トン/年、期末増減 +37%

#### 制約MSY点での種別漁獲量と期末増減（LM）
- マイワシ (x1): 11,633.6 千トン/年、期末増減 **+1261%**
- カタクチ (x2): 92.4 千トン/年、期末増減 **+4.5%**（律速種）
- ブリ (y1): 141.2 千トン/年、期末増減 +186%
- サワラ (y2): 0.5 千トン/年、期末増減 +67%

#### 重要な発見

1. **NLMでは制約MSY ≈ 無制約MSY（−2.1%）**: カタクチイワシだけが初期値割れギリギリ（+6.3%）で律速。他3種（特にマイワシ+1787%）は十分増える余裕があるため、制約がほぼ効かず、結果的に f* が無制約版とほぼ同じになる。
2. **LMでは制約MSY が顕著に低下（−16.7%）**: マイワシが急増する局面だが、カタクチだけが初期値割れ寸前（+4.5%）に達する。無制約では f_x1=0.950 のマイワシ最大獲得を狙うが、カタクチの枯渇を避けるため f_x1=0.950 を維持しつつ **f_x2 を 0.814→0.543 へ大幅削減**、**f_y2（サワラ）を 0.136→0.407 へ増加**させてバランスを取る。
3. **持続可能点が少ない（6・54 件）**: グリッド8点の粗い目盛りで、制約条件を満たす組み合わせが稀少。本来の最適点はこの feasible 領域をさらに細かく探索（2段階グリッド等）で精緻化できるが、現段階では定性的傾向を把握する段階。
4. **無制約MSYの上限張り付きは制約で解決**: NLM f_x1: 0.950→0.407, LM f_x2: 0.814→0.543・f_y2: 0.136→0.407 へと内点に引き下げられ、乱獲解が回避される。

> **結論**: 非減少制約（endpoint・4種・tol10%）は機能し、持続可能な漁獲戦略へ移行する効果を示した。ただし feasible 点の少なさから、実装の次段階では（1）2段階グリッド探索での精緻化、または（2）tol や scope の感度確認が必要。
