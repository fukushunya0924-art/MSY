# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 概要

黒潮大蛇行（LM/NLM）レジームが日本近海4魚種の資源量に与える影響を、捕食被食ODEモデルとパラメータ推定で定量化する研究コード。

- **被食者 (x)**: マイワシ (x1), カタクチイワシ (x2)
- **捕食者 (y)**: ヤリイカ (y1), スルメイカ (y2)
- **レジーム分割**: NLM 2006–2016 / LM 2017–2024

## 実行コマンド

環境は `archive/6.8_old/` 内の Poetry で管理されている。

```bash
# 依存パッケージのインストール（archive/6.8_old/ ディレクトリ内）
cd archive/6.8_old && poetry install

# 標準推定（src/ から実行）
cd src && python3 run_estimation.py            # full(16変数)モデル
cd src && python3 run_estimation.py capacity   # capacity(10変数)モデル
cd src && python3 run_estimation.py capacity_ry  # 12変数モデル

# 1位組み合わせ（ヤリイカ+スルメイカ / ブリ+サワラ）の推定
cd src/rank1_squid_largefish && python3 run_rank1.py [full|capacity|capacity_ry]

# 全魚種の組み合わせスキャン
cd src && python3 scan_species_combinations.py

# データ確認
cd src && python3 data_loader.py
```

出力 PNG は `outputs/` に保存される（例: `outputs/fit_full.png`）。

## アーキテクチャ

### ディレクトリ構成

```
data/         CSVデータ（魚種別 資源量・漁獲量時系列）
src/          現行コード（モジュール分割版）
  data_loader.py          データ読み込み・前処理・スケール統一
  model.py                ODE定義・推定エンジン（estimate関数）
  run_estimation.py       ベースライン vs 改善版の比較実行
  scan_species_combinations.py  全魚種組み合わせのランキング
  rank1_squid_largefish/  scan で1位になった組み合わせ専用の実行スクリプト
outputs/      推定結果の PNG（fit_*.png）
archive/      試行版・旧バージョン
  6.8_old/    旧バージョン（単一スクリプト、poetry環境定義を持つ）
  scripts/    開発途中の試行版スクリプト群
```

### 推定の流れ（`src/`）

1. `data_loader.load_clean_dataframe()` → 4種CSVをマージ、NaN除去
2. `data_loader.get_series()` → 全種を千トン統一、漁獲圧 `f = catch/biomass`（上限0.95）を計算
3. `run_estimation.regime_masks()` → NLM/LMの年次マスクで時系列を分割
4. `model.estimate()` → 各レジームをODE推定
   - 正規化空間（各種を全期間平均で除した平均1.0の空間）でODEを解く
   - `scipy.integrate.solve_ivp(method="LSODA")` で数値積分
   - `scipy.optimize.least_squares(method="trf")` で対数誤差を最小化
   - マルチスタート（`n_starts`個の初期値）で局所解を回避
   - 相互作用パラメータに L2 正則化（`reg_lambda`）を付与可
5. 推定後に `model._to_absolute()` で元スケールの物理パラメータへ換算

### モデル種別（`model.MODELS`）

| モデル | パラメータ数 | 特徴 | 採用方針 |
|---|---|---|---|
| `capacity` | 10変数 | r_y固定（r_y1=0.3, r_y2=0.4）、種内競争なし | 使用可 |
| `capacity_ry` | 12変数 | r_y自由化、種内競争なし | **主力モデル** |
| `full` | 16変数 | r_y自由化 + 種内競争項（A_x, A_y）あり | **使用しない** |

> **方針（2026-06-13決定）**: 今後は**環境収容力なし（種内競争項 α を含まない）モデルのみ**で進める。
> すなわち `capacity`（10変数）と `capacity_ry`（12変数）のみ対象。
> `full`（16変数）は LM 期 8点に対して過剰パラメータ・識別性崩壊・非物理的パラメータ（c1=41等）が生じるため除外。

### データスケール換算

各CSVで単位が異なるため `data_loader.get_series()` で千トンに統一している：
- マイワシ: 万トン × 10
- カタクチイワシ: 千トン（そのまま）/ 漁獲量はトン ÷ 1000
- ヤリイカ: トン ÷ 1000
- スルメイカ: 千トン（そのまま）

## 研究進捗ログ（2026-06-12〜13）

### Phase 1: ベースライン診断と推定精度の改善

#### 問題発見
archive/6.8_old/ の元コードで `least_squares` を**単一初期値・1回**だけ呼ぶと、被食者2種（マイワシ・カタクチ）だけ合わせ捕食者2種（ヤリイカ・スルメイカ）が R²<0 になる局所解に落ちることが判明した。

```
元コード ベースライン（fullモデル）
  NLM: 平均R²=-0.566  (y1=-0.357, y2=-3.733)
  LM:  平均R²=+0.167  (y1=-1.021)
```

#### 改善内容（src/ に実装）
| 改善 | 効果 |
|---|---|
| **マルチスタート最適化**（n_starts=40） | 局所解を回避。最大の効果 |
| **LSODA積分 + 許容誤差強化** | 頑健な数値積分（RK45→LSODA, rtol=1e-7） |
| **L2正則化**（reg_lambda=0.02） | 過剰適合・非識別性を緩和 |
| **実スケールR²/NRMSEの算出** | 適合度を定量的に比較可能にした |

#### 改善後の結果（src/ full モデル）
```
NLM: 平均R²=+0.500  NRMSE=0.267  (y1=+0.371, y2=-0.207)
LM:  平均R²=+0.513  NRMSE=0.235  (y1=+0.204, y2=+0.802)
```

### Phase 2: 魚種組み合わせの探索

#### 動機
LM期のカタクチイワシ(x2)とヤリイカ(y1)のフィットが悪い原因：
- **カタクチLM期**: CV=0.45、92→299と低水準での振動+急増 → ODEで追えない構造的ミスマッチ
- **ヤリイカLM期**: CPUEベースの推定量で観測誤差が大きく、上下振動が激しい

#### スキャン手法
`src/scan_species_combinations.py` で利用可能な7魚種の全有効組み合わせを評価。制約:
- **同一栄養段階グループ内**で x2種・y2種を選ぶ
- LOW（マイワシ・カタクチ）→ MID（マサバ・ヤリイカ・スルメイカ）→ HIGH（ブリ・サワラ）の順で y は x より高次

#### スキャン結果（平均CV昇順）
```
1位: ヤリイカ+スルメイカ(x) / ブリ+サワラ(y)       平均CV=0.290
2位: マサバ+ヤリイカ(x)     / ブリ+サワラ(y)       平均CV=0.310
3位: マサバ+スルメイカ(x)   / ブリ+サワラ(y)       平均CV=0.323
4位: マイワシ+カタクチ(x)   / ブリ+サワラ(y)       平均CV=0.356
現行: マイワシ+カタクチ(x)  / ヤリイカ+スルメイカ(y) 平均CV=0.472
```
ブリ(CV=0.06)・サワラ(CV=0.12)が捕食者に入るとLM期の安定性が大幅に改善する。

### Phase 3: 1位組み合わせ（ヤリイカ+スルメイカ / ブリ+サワラ）の推定

`src/rank1_squid_largefish/` に専用フォルダを作成。親の `model.py` を再利用。

#### 3モデルの比較結果（改善版・マルチスタート+正則化）

| モデル | 変数 | NLM R² | NLM NRMSE | LM R² | LM NRMSE |
|---|---|---|---|---|---|
| full | 16 | **+0.583** | **0.206** | +0.048 | 0.168 |
| capacity_ry | 12 | +0.330 | 0.244 | +0.081 | 0.163 |
| （参考）元組み合わせ full | 16 | +0.500 | 0.267 | +0.513 | 0.235 |

**NRMSEでは1位の組み合わせが両レジームとも現行より良い。**

#### 発見した構造的問題：R²とNRMSEの乖離

- LM期の R² が低い原因: ブリ(CV=0.06)はほぼ横ばいで**分散が極小** → R² = 1 - RMSE²/分散 の分母が爆発し、わずかな絶対誤差でも R² が大きな負値になる
- **平らなデータほどR²が壊れる**現象。これは適合度の問題ではなくR²の性質
- → **NRMSEを主指標として判断するのが適切**

#### 非物理的パラメータ c1 の問題

全モデルで LM 期に `c1（ブリのヤリイカ変換効率）` が異常値（full: 41.1, capacity_ry: 23.9）になる。原因：
- ブリ平均270千トン÷ヤリイカ平均10千トン = 27倍のスケール差
- 換算式 `c1 = C1 × (ブリ平均/ヤリイカ平均)` なので正規化空間での普通の値が27倍に膨らむ
- 本質的に「捕食者(ブリ)が被食者(ヤリイカ)より生物量が大きい」という生態学的逆転に起因
- 変数数の変更（16→12）では解決しない構造的問題

---

### 今後の課題と選択肢

> **モデル方針**: 環境収容力なし（α項なし）の `capacity`(10変数) / `capacity_ry`(12変数) のみ使用。`full` は除外。

1. **4位の組み合わせを試す**: マイワシ+カタクチ(x) / ブリ+サワラ(y)。被食者側は現行のまま、捕食者だけを安定種に換える。生物量逆転がなく c1 問題が解消される見込み。`capacity_ry` で走らせる。
2. **1位 vs 4位の比較**: NRMSE 主体で評価し、最終的な魚種組み合わせを確定する。
3. **指標の整理**: R² と NRMSE の乖離が起きる条件を明確化し、どの指標を論文で使うか決める。
4. **パラメータ信頼区間**: `least_squares` のヤコビアンから NLM/LM のパラメータ差の統計的有意性を評価。

---

### 適合度指標の定義（再確認用）

**RMSE**: `sqrt(mean((obs - pred)^2))` — 単位は千トン。絶対誤差。  
**NRMSE**: `RMSE / mean(obs)` — 無次元。平均に対する相対誤差。魚種をまたいで比較可能。  
**R²**: `1 - SS_res/SS_tot` — 無次元。平均横線より何%改善したか。平らなデータで負に暴れるので要注意。

→ **評価はNRMSE主体、R²は補助的に使う**のが本プロジェクトでは適切。
